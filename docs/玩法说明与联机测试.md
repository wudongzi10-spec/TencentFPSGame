# 玩法说明与联机测试

本文件说明 TencentFPSDemo 当前的玩法规则、关键 C++ 类、网络同步要点与多人自测步骤。工程基于 UE5 官方 First Person 模板的 Shooter 变体扩展，引擎版本 5.7。

## 玩法规则

- 玩家进入竞技场，服务端按波次刷新会追踪并射击玩家的敌人。
- 玩家用射击武器击败敌人，每击杀 1 个敌人，团队分 +10、个人击杀数 +1。
- 胜利条件（满足其一）：团队分达到目标分（默认 120），或清空全部波次（默认 3 波，敌人数 3/4/5）。
- 玩家死亡后延迟重生（默认 5s），在随机 PlayerStart 重新出生。

以上规则均由服务端权威判定，客户端只负责输入与显示。

## 操作

- WASD 移动，鼠标瞄准，空格跳跃。
- 左键射击，R 换弹，Q 切换武器。

## 关键 C++ 类

- 玩家：`AShooterCharacter`（血量、死亡/重生、服务端开火 RPC、武器栏）。
- 敌人：`AShooterNPC` + `AShooterAIController` + StateTree（`ShooterStateTreeUtility`：感知、面向、追踪、射击）。
- 武器/投射物：`AShooterWeapon`、`AShooterProjectile`（服务端生成与命中判定，含同阵营免伤）。
- 规则与同步：`AShooterGameMode`（服务端波次/得分/胜利）、`AShooterGameState`（团队分、波次、存活数、胜利标志、击杀提示，均复制）、`AShooterPlayerState`（个人击杀/分数）。
- UI：`AShooterHUD`（团队分、波次、生命、弹药、击杀确认、胜利面板，支持中文字体）。
- 刷怪：`AShooterNPCSpawner`（关卡出生点，向 GameMode 注册）。

## 网络同步要点

- 玩家移动：Character Movement 默认复制。
- 射击：客户端 `ServerStartFiring/ServerStopFiring` 请求服务端，服务端生成投射物并做命中判定。
- 敌人：只在服务端刷出，HP/武器/死亡状态复制到客户端。
- 分数与胜利：`TeamScore/CurrentWave/AliveEnemyCount/TargetScore/bGameWon` 等在 GameState 复制，胜利 UI 由复制状态触发。
- 阵营免伤：投射物对与开火者同阵营的角色不造成伤害（玩家 TeamByte=0，敌人 TeamByte=1），避免误伤队友和敌人互殴刷分。

## 多人联机自测步骤（PIE）

1. 用 UE5.7 打开 `TencentFPSDemo.uproject`。
2. 工具栏 Play 旁的下拉中，将 Number of Players 设为 2，Net Mode 设为 Play As Listen Server。
3. 点击 Play，进入两个玩家窗口。
4. 验证项：
   - 两名玩家可分别移动、射击，互相开火不掉血（同阵营免伤生效）。
   - 敌人自动刷新并追踪、射击玩家。
   - 击杀敌人后团队分增长、HUD 击杀确认出现、两个窗口分数一致。
   - 波次推进（第 1/2/3 波），存活数同步。
   - 达成目标分或清空波次后，两个窗口同时显示胜利面板。

## 已知前提

- AI 追踪、出生点、NavMeshBoundsVolume、StateTree 资产、各 Blueprint（玩家/敌人/武器/拾取）需在关卡中正确摆放与配置，这些为 `.uasset` 资源，不在 C++ 审查范围内，请在编辑器中确认。
- 重生后玩家需经拾取点重新获得武器（见 `progress.md` 遗留建议）。
