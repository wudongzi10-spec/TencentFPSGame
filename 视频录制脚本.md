# Demo 视频录制脚本

目标时长 2.5–4 分钟。建议 1080p、60fps 录制（OBS / Windows 录屏均可）。在编辑器里用 **2 Players + Play As Listen Server** 开两个窗口，主画面用 Server 窗口（功能最完整）。

录制前准备：
- 打开 `TencentFPSDemo.uproject`，等右下角着色器/资源编译完成，避免卡顿。
- Play 设置：Number of Players = 2，Net Mode = Play As Listen Server，勾选 New Editor Window (PIE)。
- 准备一段 30 秒左右的战斗高光留作结尾。

---

## 分镜与旁白

### 0:00–0:15　开场 / 工程来源
- 画面：编辑器里展示工程基于 UE5 First Person 模板（可快速展示 Content/Variant_Shooter 目录、Source 里的 C++ 类列表）。
- 旁白：「本 Demo 基于 UE5 官方 First Person 模板的 Shooter 变体扩展，使用 C++ 实现，目标是一个支持多人联机的据点防守 FPS。」

### 0:15–0:35　多人联机进入
- 画面：点击 Play，弹出两个玩家窗口；左右并排展示两名玩家同处一个关卡、各自移动。
- 旁白：「采用 Listen Server，两名玩家进入同一关卡，移动、射击、敌人、得分、胜利全部网络同步。」

### 0:35–1:05　基础操作与手感
- 画面：WASD 移动、鼠标瞄准、左键开火（展示开火镜头后坐力上抬后回正）、R 换弹（手臂动画）、Q 切枪、分段式弹匣随射击减少。
- 旁白：「移动、射击、换弹、切枪一应俱全；开火带镜头后坐力并自动回正，弹药用分段弹匣直观显示。」

### 1:05–1:45　敌人 AI 与战斗反馈
- 画面：敌人成波刷出、主动追踪玩家；玩家射击敌人——展示命中飘字伤害、命中标记、爆头金色标记与"爆头击杀"弹出、敌人中弹倒地（布娃娃）。
- 旁白：「敌人由 AI 控制，会追踪并射击玩家，看不到玩家时也会主动逼近。命中有飘字伤害与命中标记，参考无畏契约做了头/身/腿分段伤害，爆头近乎一枪。」

### 1:45–2:10　敌我识别与受击反馈
- 画面：敌人头顶红色血条（被掩体挡住时消失，体现无穿墙）、队友头顶绿色三角；玩家被打时屏幕红闪 + 准星周围红色方向弧指明来袭方向。
- 旁白：「敌人用红色血条标记且不穿墙，队友用绿色标记，避免误伤；受击有方向弧与屏幕红闪提示来源。」

### 2:10–2:40　得分 / 连杀 / 波次
- 画面：连续击杀触发"双杀/三杀"横幅；顶部团队分滚动累加；清空一波弹"第 N 波 已清空"，下一波弹"第 N 波 来袭"。
- 旁白：「击杀计入团队分与个人击杀/阵亡；连续击杀有连杀播报，波次推进有来袭与清空提示。」

### 2:40–3:05　死亡 / 重生 / 胜利
- 画面：玩家死亡倒地（切到另一窗口确认队友视角也是倒地）、数秒后重生并显示"重生保护"、最终达成团队目标分弹出"任务完成"结算（两个窗口同时出现）。
- 旁白：「死亡布娃娃在各端同步，重生有短暂无敌保护；团队达成目标分或清空全部波次即胜利，结算在所有客户端同步显示。」

### 3:05–3:30　收尾 / 技术点 + 链接
- 画面：快速划过几个关键 C++ 类（ShooterCharacter、ShooterNPC、ShooterGameMode/GameState、ShooterHUD），最后停在 GitHub 仓库页面或片尾卡。
- 旁白：「核心逻辑用 C++ 实现，伤害与胜利由服务端权威判定，客户端只做表现，符合多人架构。完整源码见 GitHub。」
- 片尾卡：`https://github.com/wudongzi10-spec/TencentFPSGame`

## 录制建议
- 两个窗口都录进画面，能直观证明"联机同步"；关键反馈（爆头、连杀、方向弧、倒地）尽量给特写或慢一拍。
- 如果时长紧张，优先保证：联机进入、敌人追踪与击杀、得分/连杀、死亡与重生、胜利结算。
- 旁白可后期配音，也可现场讲解；语速平稳，每个分镜一句话点题即可。
